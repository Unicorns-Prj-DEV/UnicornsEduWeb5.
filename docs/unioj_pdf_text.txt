TÀI LIỆU HƯỚNG DẪN API ĐỒNG BỘ HỆ SINH THÁI UNIOJ
Kính gửi: Các đơn vị, hệ thống đối tác tích hợp
Bộ phận công tác:  Bộ phận kỹ thuật – phát triển website UNIOJ
Phiên bản tài liệu:  1.0
Cơ sở API:  https://oj.uniedu.vn
Tài liệu này hướng dẫn các đơn vị trong hệ sinh thái Unicorns Edu kết nối với nền tảng
UNIOJ để đồng bộ tên học viên, lấy dữ liệu báo cáo cá nhân và nhúng báo cáo PDF cho phụ
huynh. API key chỉ được sử dụng trên máy chủ backend của đơn vị đối tác; không đưa ra
trình duyệt người dùng cuối.

III. API ĐỒNG BỘ TÊN (STUDENT-LOOKUP)
Dùng khi hệ thống đối tác lưu tên đầy đủ (ví dụ Hà Trần Minh Đoàn) còn UNIOJ dùng
username (ví dụ DoanHTMHCM2010) và tên hiển thị đầy đủ hơn (có thêm tỉnh, năm sinh).
STT Tham số / trường Mô tả Ví dụ
1 name (query) Tên học viên phía đối tác Hà Trần Minh Đoàn
2 username (trả về) Username UNIOJ DoanHTMHCM2010
3 display_name (trả về) Tên hiển thị trên UNIOJ Hà Trần Minh Đoàn
Hồ Chí Minh 2010
Quy tắc khớp tên: trùng khớp chính xác tên hiển thị → trùng username → tên đối tác là
tiền tố của tên UNIOJ.
curl -G "https://oj.uniedu.vn/api/student-lookup/" \
  --data-urlencode "name=Hà Trần Minh Đoàn" \
  -H "Authorization: Bearer <api_key>"

2 Server đối tác tra

VI. MÃ LỖI HTTP THƯỜNG GẶP
STT Mã Ý nghĩa
1 400 Thiếu hoặc sai tham số (ví dụ thiếu name, days không hợp lệ)
2 401 API key sai hoặc tài khoản không có quyền xem báo cáo
3 404 Không tìm thấy học viên / username
4 409 Nhiều học viên trùng tên — xem danh sách candidates trong JSON
5 503 PDF chưa bật trên môi trường (chỉ ảnh hưởng endpoint PDF)
Ghi chú: Liên hệ bộ phận kỹ thuật UNIOJ để được cấp api_key và quyền View student
personal report. Khi triển khai production, dùng URL chính thức của site UNIOJ thay cho địa chỉ
dev.
